# ПЕРВАЯ ЛЕКЦИЯ Введение в Web

## Темы курса

* Real-time сообщения (polling, websocket)
  
* Серверная разработка (node.js)

* Клиентская разработка (vanilla js)
  
* Advanced JavaScript
  
* Фундаментальный web

## Базовый сценарий работы web-приложения

1. Пользователь вводит URL.

2. Браузер загружает HTML-документ.

3. Браузер парсит HTML и загружает дополнительные ресурсы.

4. Браузер рендерит страницу.

## URL

Пример: http://mi-ami.ru:8080/profile/account.html?gender=male&age=13#comments

## HTTP

1. Протокол клиент-серверного взаимодействия.

2. Статусы ответа:

+ 1xx (информационные)

+ 2xx (успех)

+ 3xx (перенаправление)

+ 4xx (ошибка клиента)

+ 5xx (ошибка сервера)

Методы: 

+ GET - запрашивает представление указанного ресурса, запросы должны только получать данные.
  
+ POST - предназначен для отправки данных на сервер.
  
+ PUT - создаёт новый ресурс или заменяет представление целевого ресурса данными, представленными в теле запроса.
  
+ DELETE - удаляет указанный ресурс.

# ВТОРАЯ ЛЕКЦИЯ JavaScript

## Основы JavaScript

+ Переменные: let, const, var.

+ Типы данных: примитивы (string, number, boolean, symbol, null, undefined, bigint) и объекты.

+ Операторы: математические, сравнения, логические.

## Условные операторы

+ if-else - позволяет обрабатывать несколько условий, если первое условие ложно. Это расширяет возможности базовой конструкции.

+ Тернарный оператор - операция в программировании, которая возвращает одно из двух значений в зависимости от результата вычисления условного выражения. 

## Циклы

+ while - выполняет блок кода до тех пор, пока указанное условие истинно. Этот тип цикла используется, когда заранее неизвестно, сколько раз потребуется повторить выполнение блока кода.

+ do-while - выполняет блок кода хотя бы один раз, а затем проверяет условие для продолжения выполнения цикла. 

+ for - позволяет повторять блок кода определённое количество раз.

## Функции

+ Function Declaration - это объявление функции в основном потоке кода. Функция создаётся интерпретатором до выполнения кода, поэтому её можно вызвать до объявления. 
  
+ Function Expression - это объявление функции в контексте какого-либо выражения, например присваивания. Функция создаётся только когда выполнение доходит до неё, поэтому её нельзя вызвать до объявления.

+ Стрелочные функции - предоставляют более короткую запись по сравнению с обычными функциями. 

+ Замыкания - это функция, которая сохраняет доступ к своему лексическому окружению даже после того, как внешняя функция завершила выполнение.

+ Лексическое окружение - это внутренняя структура для хранения переменных, объявленных в определённой области видимости.

## Объекты и массивы

### Создание объектов

+ Литерал - Это фиксированное значение, которое не может быть изменено. 

+ Конструктор - это специальная функция, которая создаёт и инициализирует объекты. Она позволяет создавать множество объектов одного типа, используя общий шаблон.

### Методы массивов

+ push - добавляет указанные элементы в конец a и возвращает новую длину массива.

+ pop - удаляет последний элемент из массива и возвращает его. Этот метод изменяет длину метода массив.

+ shift - удаляет первый элемент из массива и возвращает этот удаленный элемент. Этот метод изменяет длину массива.

+ unshift - добавляет указанные элементы в метод и возвращает новую длину массива.

Пример:
 
![image](https://github.com/user-attachments/assets/78843faa-83d0-44aa-9271-56228713fd30)


+ slice - возвращает части массива в новый объект массива

+ splice - изменяет содержимое массива на Удаление или замена существующих элементов и/или добавление новых элементов на место.

+ map - создает новый массив, заполненный результатами вызова предоставленной функции на каждый элемент в вызывающем массиве.

+ filter - создает поверхностную копию части данного массива, отфильтрованную только до тех элементов из данного массива, которые проходят тест, реализованный предоставленной функцией.

+ reduce - передет возвращаемое значение из вычисления для предыдущего элемента. Конечным результатом выполнения редюсера по всем элементам массива является одно значение.

+ sort - сортирует элементы массива и возвращает ссылку на тот же массив, теперь отсортированный.

### Классы - содержат свойства и методы, которые определяют, как объекты этого класса будут вести себя.

+ Синтаксис классов - нужен для создания объектов с общими свойствами и методами, а также для поддержки наследования.

+ Наследование - нужно для того, чтобы создавать новые классы на основе существующих.

+ Статические методы - нужны для реализации функциональности, которая не зависит от состояния конкретного объекта.

+ Приватные свойства - нужны для инкапсуляции данных.

### Модули

+ Экспорт и импорт: (export, import) - это механизмы, которые позволяют делиться кодом между файлами и использовать экспортированные значения в других модулях.

+ Динамический импорт - позволяет загружать модули асинхронно, только когда они нужны, а не в начале файла, как при статическом импорте.

# ТРЕТЯЯ ЛЕКЦИЯ Углубленный JavaScript

## Асинхронность

Event Loop:
  
+ Макротаски - выполняются после завершения текущей синхронной задачи. Эти задачи обрабатываются в порядке очереди, одна за другой.

+ Микротаски - выполняются сразу после завершения текущей синхронной задачи, но перед началом следующей макрозадачи.

Промисы: 

+ then - сохраняет обратные вызовы в промисе, для которого он вызывается, и немедленно возвращает другой объект.
   
+ catch - немедленно возвращает другой объект promise, что позволяет цепочно вызывать другие методы.
  
+ finally - позволяет выполнить код после того, как промис завершил своё выполнение, независимо от того, был ли он разрешён или отклонён.

+ Promise.all -  ожидает выполнения всех промисов и возвращает массив их результатов.  

+ Promise.race - ждёт первый завершившийся промис из списка и передаёт его результат дальше.

+ Async/await - это синтаксический сахар для работы с промисами, который позволяет писать асинхронный код так, как если бы он был синхронным.

## Обработка ошибок

+ try-catch-finally - это конструкция позволяет обрабатывать ошибки во время выполнения программы. Она позволяет определить блок кода, в котором могут возникать ошибки, и задать способ их обработки.

+ Пользовательские ошибки - ошибки которые были совершенны юзером.

## Регулярные выражения

Синтаксис: new RegExp.

+ Шаблон - строка, содержащая регулярное выражение.

+ Флаги - необязательная строка, которая определяет поведение поиска.

Методы:
+ test - который тестирует совпадение в строке. Возвращает либо истину либо ложь.
  
+ exec - который выполняет поиск совпадения в строке. Он возвращает массив данных.
  
+ match -который выполняет поиск совпадения в строке. Он возвращает массив данных либо null если совпадения отсутствуют.
  
+ replace - который выполняет поиск совпадения в строке, и заменяет совпавшую подстроку другой подстрокой, переданной как аргумент в этот метод.
